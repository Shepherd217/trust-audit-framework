## Research Conducted

### 1. Supabase TypeScript Best Practices
- Use `npx supabase gen types typescript` to generate `database.types.ts` from actual schema
- Import types: `import { Database } from './database.types'`
- Type client: `createClient<Database>(...)`
- Helper types for tables: `Tables<T>`, `Enums<T>`, `QueryData<T>`
- Automated CI syncing via GitHub Actions

### 2. BigInt ES2020 Fix
- Error: "Big integer literals not available in configured target environment"
- Root cause: `tsconfig.json` target < ES2020
- Fix: `"target": "ES2020"` in compilerOptions
- Alternative: `"lib": ["es2020.bigint"]` while keeping lower target

### 3. CI/CD Best Practices
- Use `npm ci` not `npm install` in CI (deterministic, faster)
- `npm ci` requires `package-lock.json` committed
- Without lockfile: npm resolves dependencies fresh every build (~10+ min)
- With lockfile + npm ci: ~1 min builds

### 4. TypeScript Strict Mode (Next.js 15)
- `"strict": true` catches errors early
- Additional flags: `noUncheckedIndexedAccess`, `noImplicitOverride`
- Next.js 15 + Turbopack = 700x faster cold starts
- Use App Router + React Server Components
