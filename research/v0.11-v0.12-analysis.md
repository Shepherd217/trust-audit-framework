## 1. Vector Analysis: Overturn Direction

### The Question
When a verdict is overturned on appeal, **which direction does it move?**

### Direction Vectors

| Direction | Definition | Interpretation |
|-----------|------------|----------------|
| `toward_expertise` | New committee has HIGHER expertise score than original | Appeal corrected toward specialists |
| `toward_majority` | New committee has LOWER expertise score than original | Appeal regressed to non-specialists |
| `neutral` | Equal expertise, or same outcome | No directional change |

### Implementation

```sql
-- Calculated in expertise_history table
overturn_direction = CASE
    WHEN new_expertise > original_expertise THEN 'toward_expertise'
    WHEN new_expertise < original_expertise THEN 'toward_majority'
    ELSE 'neutral'
END
```

### Expected Patterns (Hypothesis)

| Scenario | Expected Direction | Rationale |
|----------|-------------------|-----------|
| Technical disputes | `toward_expertise` | Specialists correct generalists |
| Behavioral disputes | `neutral` or `toward_majority` | Social consensus matters |
| Novel domains | `toward_expertise` | Domain experts prevail |

## 2. Expertise Distance Metric

### The Question
**How far** does expertise shift on appeal?

### Calculation

```
expertise_distance = new_committee_expertise - original_committee_expertise
```

### Range Interpretation

| Distance | Meaning |
|----------|---------|
| +1000 to +5000 | Major shift toward expertise |
| +100 to +999 | Moderate shift toward expertise |
| -100 to +100 | Minimal shift (neutral zone) |
| -999 to -100 | Moderate shift away from expertise |
| -5000 to -1000 | Major shift away from expertise |

### Quality Thresholds

```sql
-- Quality classification (after durability period)
overturn_quality = CASE
    WHEN re_dispute_filed = false AND overturn_direction = 'toward_expertise' 
        THEN 'improvement'
    WHEN re_dispute_filed = true AND overturn_direction = 'toward_majority' 
        THEN 'regression'
    ELSE 'neutral'
END
```

## 3. Calibration Tracking (ECE/Brier)

### Expected Calibration Error (ECE)

**Definition:** When an agent reports 70% confidence, are they correct 70% of the time?

```
ECE = Σ (bin_count / total) × |predicted_accuracy - actual_accuracy|
```

**Interpretation:**
- 0.000 = Perfectly calibrated
- 0.100 = Good calibration
- 0.250 = Poor calibration
- 0.500+ = Miscalibrated

### Brier Score

**Definition:** Accuracy of probabilistic predictions

```
Brier = (1/N) × Σ (predicted - outcome)²
```

**Interpretation:**
- 0.00 = Perfect
- 0.25 = Random guessing (binary)
- 2.00 = Worst possible

### Weekly Calibration Report Structure

```json
{
  "week": "2026-03-17",
  "agent_calibrations": [
    {
      "agent_id": "agent_abc123",

      "ece_score": 0.083,

        "0.5": {"predicted": 0.50, "actual": 0.48, "count": 4},
```

## 4. Durability Metrics (v0.12.1 Preview)

### Re-Dispute Rate

**Definition:** After an overturn, how often does the same disputant file again?

```
re_dispute_rate = re_dispute_count / total_overturns
```

**Interpretation:**
- Low rate (< 10%): Overturns are durable
- High rate (> 30%): Overturns don't satisfy disputants

### Re-Correction Rate

**Definition:** After an overturn, how often does THAT verdict get overturned again?

```
re_correction_rate = re_correction_count / total_overturns
```

**Interpretation:**
- Low rate (< 5%): Appeals are final
- High rate (> 20%): Unstable resolution

### Decision Durability Score

**Composite formula:**

```
durability_score = 
    (1 - re_dispute_rate) × 40 +
    (1 - re_correction_rate) × 40 +
    improvement_rate × 20
```

**Range:** 0-100

| Score | Rating |
|-------|--------|
| 90-100 | Excellent durability |
| 70-89 | Good durability |
| 50-69 | Fair durability |
| < 50 | Poor durability |

## 6. SQL Queries for Analysis

### Overturn Direction Distribution

```sql
SELECT 
    overturn_direction,
    COUNT(*) as count,
    AVG(expertise_distance) as avg_distance,
    AVG(CASE WHEN re_dispute_filed THEN 1 ELSE 0 END) as re_dispute_rate
FROM expertise_history
WHERE overturned_at IS NOT NULL
GROUP BY overturn_direction;
```

### Calibration by Tier

```sql
SELECT 
    calibration_tier,
    COUNT(*) as agent_count,
    AVG(ece_score) as avg_ece,
    AVG(brier_score) as avg_brier
FROM agent_calibration
WHERE week_start = '2026-03-17'
GROUP BY calibration_tier;
```

### Durability Trends

```sql
SELECT 
    week_start,
    overturn_rate,
    re_dispute_rate,
    re_correction_rate,
    durability_score
FROM durability_metrics
ORDER BY week_start;
```
