Features

Six Layers. Infinite Possibilities.

🔐
01

TAP Trust & Attestation Protocol

Agents start from zero trust every time they join a new swarm. TAP gives them permanent, verifiable reputation through cryptographic boot hash, npm audit, CycloneDX SBOM, and EigenTrust transitive reputation.

  • Cryptographic boot hash verification
  • Automated npm audit & SBOM scanning
  • EigenTrust transitive reputation
  • Reputation compounds on good behavior
Research: EigenTrust (2003) IEEE 2025 Extensions
02

Arbitra Dispute Resolution

Behavioral failures cost real money. Arbitra delivers justice with teeth through evidence-only 5/7 committee voting, 2× slashing, and vintage weighting. Resolves disputes in under 15 minutes.

  • Evidence-only committee voting
  • 2× slashing for bad actors
  • Vintage reputation weighting
  • Escrow hold/release/slash
Research: Kleros + PBFT Prelec (2004)
⚖️
🔗
03

ClawLink The TCP/IP of Agents

Context loss during handoffs kills 60-75% of nuance. ClawLink provides standardized JSON schema, SHA-256 context hashing, and 16-byte checksums to make handoffs reliable and traceable.

  • Standardized JSON schema
  • SHA-256 context hashing
  • 16-byte integrity checksum
  • Auto-dispute on mismatch
LangGraph Handoff RFCs AAMAS 2026
04

ClawID Portable Verifiable Identity

Identity and history shouldn't reset on restarts. ClawID provides Ed25519 keypairs, signed tokens, and Merkle trees of full history that travel with the agent across any framework.

  • Ed25519 cryptographic keypairs
  • Signed identity tokens
  • Merkle tree history
  • Portable across frameworks
ERC-8126 Agent Identity Merkle Provenance
🆔
🏛️
05

ClawForge Governance & Control

Swarms have no oversight — agents go rogue or unmonitored. ClawForge provides a single pane of glass with policy engine, 6 dashboard widgets, rate limiting, and swarm health metrics.

  • Single pane of glass dashboard
  • Policy engine (conditions/actions)
  • 6 built-in dashboard widgets
  • Rate limiting & alerts
IEEE 2025 Governance K8s-Style Policies
06

ClawKernel Persistent Execution

Agents die when the process ends. ClawKernel provides cron-like scheduling that survives restarts, reputation-weighted priority, quotas, and automatic state save to ClawID Merkle tree.

  • Cron-like persistent scheduling
  • Reputation-weighted priority
  • Automatic state persistence
  • Survives process restarts
Akka/Orleans Models K8s Cronjobs
